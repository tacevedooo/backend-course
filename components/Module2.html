Node.js y Express

¿Qué es Node.js?

Node.js es un entorno de ejecución de JavaScript (runtime) orientado a eventos y diseñado para construir aplicaciones de red escalables. A diferencia del JavaScript tradicional que corre en el navegador, Node.js permite ejecutar código en el servidor. Su característica principal es que es asíncrono y no bloqueante, lo que significa que puede manejar miles de conexiones simultáneas sin detenerse a esperar que una tarea (como leer un archivo o consultar una base de datos) termine para empezar la siguiente.

¿Cómo funciona la arquitectura de Node.js?

A diferencia de los servidores tradicionales que asignan un hilo de memoria por cada usuario, Node.js opera con un hilo único principal que nunca se detiene a esperar. Su arquitectura se basa en un bucle de eventos (Event Loop) que actúa como un gestor de tráfico: recibe todas las peticiones y, si alguna requiere una tarea pesada (como leer un disco o una base de datos), la delega al sistema operativo o a un pool de hilos interno mediante I/O no bloqueante. Esto permite que el hilo principal quede libre de inmediato para seguir atendiendo a otros usuarios, retomando la tarea anterior mediante un callback solo cuando el resultado está listo, logrando así una alta escalabilidad con un consumo mínimo de recursos.

Del HTTP puro a Express.js

Aunque el módulo http de Node.js es potente, funciona a muy bajo nivel. En aplicaciones reales, esto presenta dificultades críticas:

  • Gestión de rutas manual: Requiere escribir estructuras if/else o switch gigantes para validar cada URL y método (GET, POST, etc.).
  • Parseo complejo: Extraer datos del cuerpo (JSON) o parámetros de la URL implica escribir mucho código repetitivo y propenso a errores.
  • Código verboso: Tareas simples, como enviar archivos o gestionar errores, ensucian la lógica principal del archivo.
  • Escalabilidad limitada: A medida que el proyecto crece, el código se vuelve difícil de mantener sin una estructura de framework.

¿Qué es Express.js?

Express es el framework web estándar para Node.js. Se define como un marco "minimalista y flexible" que añade una capa de herramientas sencillas sobre las funcionalidades nativas de Node, sin ocultar sus características principales.

¿Por qué se usa Express?

  • Middleware: Permite ejecutar funciones intermedias (logueo, seguridad, validación) de forma organizada.
  • Enrutamiento sencillo: Define rutas claras como app.get('/usuarios', ...) en lugar de analizar strings de URL.
  • Productividad: Reduce drásticamente la cantidad de código repetitivo (boilerplate) necesario para levantar una API.
  • Ecosistema: Es compatible con miles de paquetes y tiene la comunidad más grande de Node.js.

Instalación

Para comenzar un proyecto con Express debes instalar el paquete vía npm:

Terminal
# Instalar Express
npm install express

Tu primer servidor

Una vez instalado, crea un archivo index.js y añade el siguiente código base:

index.js
// 1. Importar express
const express = require('express');

// 2. Inicializar la aplicación
const app = express();
const port = 3000;

// 3. Definir una ruta (Endpoint)
app.get('/', (req, res) => {
  res.send('¡Hola Mundo desde Express!');
});

// 4. Escuchar peticiones en el puerto 3000
app.listen(port, () => {
  console.log(`Servidor corriendo en http://localhost:${port}`);
});

Cómo encender y probar tu servidor

Para ver tu código en acción, sigue estos pasos desde la terminal de tu proyecto:

Terminal
# Ejecuta el archivo principal
node index.js

Si ves el mensaje Servidor corriendo en http://localhost:3000, ¡ya está al aire!

Probar en el navegador

Abre tu navegador (Chrome, Firefox, etc.) y escribe en la barra de direcciones:
http://localhost:3000

💡 Tip Pro: Para detener el servidor en la terminal, presiona Ctrl + C. Recuerda que debes reiniciarlo cada vez que hagas un cambio en el código para que surta efecto.

Routing en Express

El routing en Express es la forma en que definimos qué debe hacer el servidor cuando recibe una petición en una URL específica.

En otras palabras, el routing permite decirle al backend:

“Si alguien entra a esta ruta, responde esto”

Ejemplo básico

index.js
app.get("/", (req, res) => {
  res.send("Hola mundo");
});

Esto significa:

  • Cuando alguien visite http://localhost:3000/
  • El servidor responde: "Hola mundo"

Tipos de rutas más comunes

Ejemplos de Métodos HTTP
app.get("/users", (req, res) => {
  res.send("Obtener usuarios");
});

app.post("/users", (req, res) => {
  res.send("Crear usuario");
});

app.put("/users/:id", (req, res) => {
  res.send("Actualizar usuario");
});

app.delete("/users/:id", (req, res) => {
  res.send("Eliminar usuario");
});

Middlewares en Express

Un middleware es simplemente una función que se ejecuta en medio de una petición (Request) y una respuesta (Response). Tienen acceso al objeto de la petición, al objeto de la respuesta y a la siguiente función de middleware en el ciclo de vida de la solicitud.

Imagina que el middleware es un punto de control o una aduana: puede revisar si el usuario tiene permiso para pasar, transformar los datos que llegan, o simplemente registrar qué está pasando en el servidor.

1. Middleware Global

Se define con app.use() y se ejecuta para todas las rutas de tu aplicación. Es ideal para tareas como logueo de peticiones o configuración de seguridad.

index.js
// Este código se ejecuta en cada petición
app.use((req, res, next) => {
  console.log(`Nueva petición recibida: ${req.method} ${req.url}`);
  
  // ¡Importante! Pasar al siguiente paso
  next(); 
});

2. Middleware de Ruta

A veces solo necesitas que un middleware se ejecute en una ruta específica (por ejemplo, para revisar si alguien está logueado antes de dejarlo ver su perfil).

index.js
const checkAuth = (req, res, next) => {
  console.log("Revisando autenticación...");
  next();
};

// Solo se ejecuta checkAuth cuando entran a /dashboard
app.get("/dashboard", checkAuth, (req, res) => {
  res.send("Bienvenido a tu panel");
});
