Módulo 3

Base de Datos (SQL y MongoDB)

¿Por qué necesitamos una Base de Datos?

En el módulo anterior, si reiniciabas tu servidor, todos los usuarios nuevos desaparecían. Esto sucede porque los datos se guardaban en la memoria RAM (volátil). Las Bases de Datos permiten la persistencia: los datos se graban en el disco y permanecen allí aunque el servidor se apague.

Bases de datos estructuradas (SQL)

Organizan la información en tablas con filas y columnas. Tienen una estructura fija (esquema) y son ideales cuando la consistencia es prioridad.

Ejemplo estructurado (Tabla Usuarios)
| id | nombre  | email            | telefono  |
|----|---------|------------------|-----------|
| 1  | Tomás   | tomas@email.com  | 555-0123  |
| 2  | Lucía   | lucia@email.com  | NULL      |
| 3  | Andrés  | andres@email.com | 555-9876  |

Bases de datos NO estructuradas (MongoDB)

Organizan la información en documentos (JSON). Tienen una estructura flexible; cada registro puede tener campos distintos sin afectar a los demás.

Ejemplo No estructurado (Colección Usuarios)
[
  { "name": "Tomás", "email": "tomas@email.com", "tel": "555-0123" },
  { "name": "Lucía", "email": "lucia@email.com" },
  { "name": "Andrés", "email": "andres@e.com", "premium": true }
]

¿Qué base de datos usar en cada caso?

La elección depende de la naturaleza de tus datos y la velocidad de desarrollo que necesites:

Guía de Decisión Rápida
| Escenario                    | Recomendación | ¿Por qué?                     |
|------------------------------|---------------|-------------------------------|
| Datos fijos y relaciones     | SQL (MySQL)   | Estructura clara y segura.    |
| Prototipos y apps rápidas    | NoSQL (Mongo) | Flexibilidad total de campos. |
| Gran volumen / Escalamiento  | NoSQL (Mongo) | Crecimiento horizontal fácil. |
| Datos financieros / Complejos| SQL (Postgres)| Integridad y consistencia.    |

Un recordatorio vital: ¿Qué es CRUD?

Independientemente del tipo de base de datos que elijas, el 90% de tu trabajo como desarrollador backend consistirá en gestionar las cuatro operaciones fundamentales de los datos:

  • C (Create): Crear o insertar nuevos registros/documentos.
  • R (Read): Leer o consultar datos existentes.
  • U (Update): Actualizar o modificar información.
  • D (Delete): Eliminar registros.

Ahora veremos cómo Node.js implementa estas operaciones en ambos mundos.

SQL con Node.js

Para trabajar con SQL (en este caso MySQL), necesitamos un "driver" o librería que nos permita enviar comandos SQL desde nuestro código JavaScript. Usaremos la librería mysql2.

Instalación
npm install mysql2
Ejemplo básico (MySQL)
import mysql from 'mysql2/promise';

// 1. Configuración del pool de conexiones
const pool = mysql.createPool({
  host: 'localhost',
  user: 'root',
  password: '', // Tu contraseña de MySQL
  database: 'test', // Asegúrate de tener una base de datos llamada 'test' creada en tu MySQL
});

// 2. Función para inicializar la base de datos
const initializeDatabase = async () => {
  try {
    // SQL para crear la tabla si no existe
    const createTableQuery = `
      CREATE TABLE IF NOT EXISTS users (
        id INT AUTO_INCREMENT PRIMARY KEY,
        name VARCHAR(255) NOT NULL
      )
    `;
    await pool.query(createTableQuery);
    console.log("Tabla verificada");
  } catch (error) {
    console.error("Error al inicializar tabla:", error.message);
  }
};

// 3. Función para consultar todos los usuarios
const getAllUsers = async () => {
  try {
    // Retorna un arreglo donde el primer elemento son las filas (rows)
    const [rows] = await pool.query("SELECT * FROM users");
    return rows;
  } catch (error) {
    console.error("Error al consultar usuarios:", error.message);
    throw error;
  }
};

// Ejecución del flujo
const start = async () => {
  await initializeDatabase();
  
  const users = await getAllUsers();
  console.log("Listado de usuarios:", users);
};

start();

MongoDB con Node.js

Para MongoDB, el enfoque es más "orientado a objetos" u "orientado a documentos". En lugar de escribir SQL puro, usaremos la librería mongoose, que actúa como un mapa entre nuestro código JS y los documentos de MongoDB, permitiéndonos definir "modelos" de datos.

Instalación
npm install mongoose
Ejemplo con Mongoose
import mongoose from "mongoose";

// 1. Conectar a la base de datos
mongoose.connect("mongodb://localhost:27017/test")
  .then(() => console.log("Conectado a MongoDB"))
  .catch(err => console.error("Error de conexión", err));

// 2. Definir el Modelo (Esquema de un usuario)
const User = mongoose.model("User", {
  name: String,
  email: String
});

// 3. Crear un nuevo usuario (CREATE)
const user = new User({
  name: "Tomás",
  email: "tomas@email.com"
});

// 4. Guardar en la base de datos
user.save()
  .then(() => console.log("Usuario guardado con éxito"))
  .catch(err => console.error("Error al guardar", err));

Actividad Práctica: Persistencia Real

Es hora de evolucionar tu proyecto del Módulo 2. Ya no usaremos let users = [...]. Ahora, tus usuarios vivirán en una base de datos real. Debes seleccionar entre MongoDB o MySQL (o cualquier otra base de datos que prefieras) y migrar tu código para que las operaciones CRUD interactúen con esa base de datos en lugar de un arreglo en memoria.

Paso 1: Preparación del entorno

Asegúrate de tener el motor de base de datos corriendo en tu PC e instala la librería necesaria en tu terminal:

Terminal
# MongoDB
npm install mongoose

# MySQL
npm install mysql2

Paso 2: Configurar la conexión

Opción 1: MongoDB (NoSQL)

index.js
import mongoose from 'mongoose';

mongoose.connect('mongodb://localhost:27017/test')
  .then(() => console.log("¡Base de datos conectada!"))
  .catch(err => console.error("Fallo de conexión", err));

// Modelo
const User = mongoose.model('User', {
  name: String
});

Opción 2: MySQL (SQL)

index.js
import mysql from 'mysql2/promise';

// 1. Crear el pool de conexiones con soporte para promesas
const connection = mysql.createPool({
  host: 'localhost',
  user: 'root',
  password: '', // Tu contraseña de MySQL
  database: 'test',
  waitForConnections: true,
  connectionLimit: 10,
  queueLimit: 0
});

// 2. Función para inicializar la tabla
const setupDatabase = async () => {
  try {
    const createTableQuery = `
      CREATE TABLE IF NOT EXISTS users (
        id INT AUTO_INCREMENT PRIMARY KEY,
        name VARCHAR(255) NOT NULL
      )
    `;
    // Ejecuta la creación de la tabla
    await connection.query(createTableQuery);
    console.log("Tabla 'users' verificada o creada");
  } catch (error) {
    console.error("Error al inicializar la tabla:", error.message);
  }
};

Concepto Importante: async / await

Cuando trabajamos con bases de datos, las operaciones no son instantáneas. Consultar, insertar o eliminar datos toma tiempo, ya que implica comunicación con otro sistema (la base de datos). Por eso usamos async y await.

Paso 3: Migrar los Endpoints (CRUD)

MongoDB (Mongoose)

index.js
// GET
app.get('/users', async (req, res) => {
  const users = await User.find();
  res.json(users);
});

// POST
app.post('/users', async (req, res) => {
  const newUser = new User({ name: req.body.name });
  await newUser.save();
  res.status(201).send("Usuario creado");
});

// PUT
app.put('/users/:id', async (req, res) => {
  await User.findByIdAndUpdate(req.params.id, { name: req.body.name });
  res.send("Usuario actualizado");
});

// DELETE
app.delete('/users/:id', async (req, res) => {
  await User.findByIdAndDelete(req.params.id);
  res.send("Usuario eliminado");
});

MySQL

index.js
// GET
app.get('/users', async (req, res) => {
  const [rows] = await connection.query('SELECT * FROM users');
  res.json(rows);
});

// POST
app.post('/users', async (req, res) => {
  await connection.query('INSERT INTO users (name) VALUES (?)', [req.body.name]);
  res.status(201).send("Usuario creado");
});

// PUT
app.put('/users/:id', async (req, res) => {
  await connection.query('UPDATE users SET name = ? WHERE id = ?', 
    [req.body.name, req.params.id]);
  res.send("Usuario actualizado");
});

// DELETE
app.delete('/users/:id', async (req, res) => {
  await connection.query('DELETE FROM users WHERE id = ?', [req.params.id]);
  res.send("Usuario eliminado");
});

Paso 4: Prueba tu servidor

Para probar que tu servidor funciona, no basta con usar el navegador. Necesitas un Cliente API para enviar datos en formato JSON y probar los métodos POST, PUT y DELETE. Acá te dejo una recomendación:

  1. Enciende tu servidor: node index.js.
  2. Crea un usuario con POST.
  3. Apaga el servidor.
  4. Vuelve a encenderlo y haz un GET.
